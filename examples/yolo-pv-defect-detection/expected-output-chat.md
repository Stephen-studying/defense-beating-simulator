## 高风险缺口（最多 5 项）

1. 数据集来源、授权和划分缺失，指标可信度无法判断。
2. baseline 对比没有完整结果和公平实验条件。
3. 缺少消融实验，无法证明改进模块带来提升。
4. 指标没有对应阈值、类别结果和失败案例。
5. “参与”范围不够具体，容易被追问个人代码和实验所有权。

## 连续追问链

### 保研导师：数据集

- Q1：数据集来自哪里，是否公开或获得授权？
- Q2：训练、验证和测试集如何划分？
- Q3：如何避免同一组件或连续帧跨集合造成数据泄漏？
- Risk：High
- Evidence basis：输入材料明确说数据集说明缺失。

### 严苛审稿人：模型改进

- Q1：具体修改了 YOLO 的哪个模块？
- Q2：该设计是原创、复现论文，还是引用开源实现？
- Q3：在相同数据划分和训练配置下，baseline 与各消融项分别是多少？
- Risk：High
- Evidence basis：材料只说有改进和对比，未提供完整证据。

## 防守型回答策略

当前项目是在 YOLO baseline 基础上的光伏缺陷检测实验。我实际参与了数据整理、训练配置、改进实验和结果分析，但需要进一步按具体模块说明个人工作。现有材料还不足以证明改进稳定有效或具有原创性，因此不能只凭一组指标宣称全面提升。下一步应先补齐数据集与划分，再完成同条件 baseline、消融实验和失败案例分析。

## 材料修复清单

1. README 补数据集来源、授权、类别分布和划分策略。

3. PPT 增加误检、漏检和典型失败案例。

5. 把个人贡献细化到数据、训练、模型、分析或系统模块。
